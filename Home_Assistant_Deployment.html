Home Assistant Deployment

Home Assistant by Hass.io has several ways the software can be deployed. I tested a variety of ways to deploy Home Assistant, but chose to leverage the Docker containerized deployment method.

Getting Started - What do you Need


I chose to leverage Ubuntu as my operating system of choice. Home Assistant supports a variety of operating systems and deploymnet methods. More can be found here.

How did I get started?


The first thing I determined is where I wanted to run Home Assistant. After my testing, I determined a Raspberry Pi didn't make long term sense and the reads and writes required for Home Assistant would destroy the micro sd card in the Pi. I had an old server laying around that I used to run vmWare on for a home lab and it more than met the minimum specs required to run Home Assistant.

What are those Specs?

Type Minimum
Storage 32 GB
Memory 1 GB
Network 100 Mb/s wired
After deciding what hardware I wanted to run Home Assistant on long term, I needed an installer for Ubuntu's latest LTS (Long Term Support) version. Currently, Ubuntu's latest LTS version is 20.04. You can download it here. After downloading Ubuntu, I used (Balena Etcher on my Mac OS to write the files to a USB for installtion.
